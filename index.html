☁ Synchronisation cloud — Supabase
Configuration en 3 étapes
1. Compte gratuit sur supabase.com → nouveau projet
2. SQL Editor → copier-coller et exécuter ce script :
-- Créer la table CREATE TABLE IF NOT EXISTS vignette_data ( id TEXT PRIMARY KEY, data TEXT NOT NULL, updated_at TIMESTAMPTZ DEFAULT now() ); -- Activer RLS ALTER TABLE vignette_data ENABLE ROW LEVEL SECURITY; -- Lecture : uniquement la ligne 'main' CREATE POLICY "read_main" ON vignette_data FOR SELECT TO anon USING (id = 'main'); -- Insertion : uniquement la ligne 'main' CREATE POLICY "insert_main" ON vignette_data FOR INSERT TO anon WITH CHECK (id = 'main'); -- Mise à jour : uniquement la ligne 'main' CREATE POLICY "update_main" ON vignette_data FOR UPDATE TO anon USING (id = 'main') WITH CHECK (id = 'main'); ⚠ Ne jamais créer de politique DELETE — les données ne peuvent pas être supprimées de l'extérieur.
3. Project Settings → API → copier l'URL et la clé anon public
Si vous aviez déjà l'ancienne politique — supprimez-la d'abord dans Supabase :
DROP POLICY IF EXISTS "anon_all" ON vignette_data;
Puis exécutez le script ci-dessus.
